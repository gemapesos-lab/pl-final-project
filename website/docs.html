ToyLang Reference

ToyLang is a minimal imperative language designed for learning compiler construction. It is intentionally small — just integers, four arithmetic operators, and simple I/O — so you can focus on the structure of a language implementation rather than its complexity.

Pure Python ~600 lines Integer-only Recursive descent Tree-walk interpreter .tl extension
🧸
Introduction
What is ToyLang and why does it exist?

ToyLang is a front-end compiler and tree-walking interpreter implemented in pure Python 3.10+. The entire implementation — lexer, parser, semantic analyser, and interpreter — fits in roughly 600 lines of code, making it an ideal study case for understanding how programming language tools work from first principles.

The language is deliberately constrained: it supports only integer variables, basic arithmetic, and stdin/stdout I/O. There are no strings, no booleans, no control flow. This keeps the grammar small enough to reason about completely while still exercising every phase of a typical compiler pipeline.

💡
ToyLang source files use the .tl file extension.

Quick Start
Running your first ToyLang program

Hello, Numbers!

Save the following as hello.tl:

toylang 
var x;

output x;

Run it with the Python CLI:

shell 
./main.py hello.tl

Output: 42

Identifiers
Naming rules for variables

An identifier is a name used to refer to a variable. ToyLang identifiers are ASCII-only and must follow these rules:

  • May contain ASCII letters (a–z, A–Z), digits (0–9), and underscores (_).
  • Must not start with a digit.
  • Must not contain non-ASCII characters (e.g. accented letters, emoji).
  • Are case-sensitive: count and Count are different names.

Integer Literals
How to write numbers in ToyLang

Integer literals are written as sequences of decimal digits (0–9). There are no negative literals — use the unary minus operator to negate: -5.

✅ Valid literals❌ InvalidReason
0, 1, 42, 1000-42 as a literalUse -42 as a unary expression instead
Any non-negative integer3.14Floats are not supported
0xFFOnly decimal notation
💡
Values are stored using Python's arbitrary-precision integers internally, but extremely large literals (beyond Number.MAX_SAFE_INTEGER) will trigger a lexer overflow error in the web playground.

Comments
Block comments, the only comment style

ToyLang supports only block comments using /* ... */ syntax. There are no line comments (// is not valid).

Block comments may:

  • Span multiple lines
  • Appear anywhere whitespace is allowed (before, between, or after tokens)
  • Be used inline within a statement
⚠️
Block comments cannot be nested. /* outer /* inner */ end */ — the first */ closes the comment, leaving end */ as unexpected tokens.
toylang 
/* This is a block comment */
var /* inline comment */ total;

/*
  Multi-line comment.
  Explains the logic below.
*/
total = (8 + 4) / 3;
output total; /* outputs 4 */
📦
Variables
Declaring and using integer variables

All variables in ToyLang are integers. There is only one type.

Declaration

Every variable must be declared with var before it can be used. Declaration initialises the variable to 0.

toylang 
var x;      /* x is now 0 */
var count;  /* count is now 0 */

Scope

There is no block scoping — all variables exist for the entire program.

Rules

  • A variable must be declared before it is read or assigned.
  • A variable cannot be declared twice (duplicate declaration is a semantic error).
  • Multiple declarations are allowed as long as each uses a unique name.
❌
Use before declaration:
output x; var x; — semantic error: variable 'x' used before declaration
❌
Duplicate declaration:
var x; var x; — semantic error: duplicate declaration for 'x'
📋
Statements
The four statement types in ToyLang

Every statement ends with a semicolon ;. ToyLang has exactly four statement types:

1. Variable Declaration — var

syntax 
var IDENTIFIER;

Declares a new integer variable, initialised to 0. The identifier must be unique (not previously declared).

2. Assignment

syntax 
IDENTIFIER = expression;

Evaluates the expression and stores the result in the (already-declared) variable.

toylang 
var x;
x = 2 + 3 * 4;  /* x = 14 */
x = x + 1;        /* x = 15 */

3. Input — input

syntax 
input IDENTIFIER;

Reads one line from standard input, parses it as an integer, and stores it in the variable. The runtime prompts with name = before reading.

⚠️
If the input is not a valid integer, or if stdin is exhausted, a runtime error is raised.

4. Output — output

syntax 
output IDENTIFIER;

Prints the current value of the variable as a decimal integer followed by a newline to standard output.

Integer Division

The / operator performs integer division truncated toward zero — the same as C's / operator for integers.

ExpressionResultNote
7 / 23Truncates, doesn't round
-7 / 2-3Toward zero, not floor (floor would be -4)
7 / -2-3Same sign logic
5 / 0runtime errorDivision by zero

Unary Minus

The unary minus - negates its operand. It is right-associative and binds tighter than multiplication:

toylang 
var x;
x = -5;          /* x = -5 */
x = --3;         /* x = 3  (double negation) */
x = 3 - -2;      /* x = 5  (3 minus negative 2) */
output x;

Parentheses

Use parentheses to override default precedence:

toylang 
var a;
var b;
a = 2 + 3 * 4;    /* a = 14  (3*4 first) */
b = (2 + 3) * 4;  /* b = 20  (2+3 first) */
output a;
output b;

1

Worked Example

toylang 
var x;
x = 3 - -2 * (4 + 1);
output x;

Evaluation order:

  1. (4 + 1) = 5 — parentheses first
  2. -2 = -2 — unary minus
  3. -2 * 5 = -10 — multiplication
  4. 3 - (-10) = 13 — subtraction

Output: 13

Compiler Pipeline
How source code becomes results

ToyLang processes source code in four sequential phases:

#PhaseInputOutputCatches
1 🔤 Lexer Source text (UTF-8) Token stream Invalid characters, malformed literals, unterminated comments
2 🌳 Parser Token stream Abstract Syntax Tree (AST) Missing keywords, mismatched delimiters, unexpected tokens
3 🔍 Semantic Analyser AST Validated AST (or error) Use-before-declaration, duplicate declarations
4 ⚡ Interpreter Validated AST Program output Division by zero, bad/missing input
💡
Each phase runs only if the previous phase succeeds. A lexer error prevents parsing; a parser error prevents semantic analysis; and so on.

The Playground Tabs

The playground's four tabs correspond directly to these phases:

  • Tokens — output of the Lexer (phase 1)
  • AST — output of the Parser (phase 2)
  • Semantic — result of the Semantic Analyser (phase 3)
  • Output — result of the Interpreter (phase 4)

Formal Grammar (BNF)
The complete syntactic specification of ToyLang

ToyLang's grammar is a simple LL(1) grammar, parsed by a hand-written recursive descent parser. Each non-terminal maps directly to a method in the parser:

program := statement* EOF statement := "var" IDENT ";" | IDENT "=" expr ";" | "input" IDENT ";" | "output" IDENT ";" expr := term ( ( "+" | "-" ) term )* term := unary ( ( "*" | "/" ) unary )* unary := "-" unary | primary primary := INT | IDENT | "(" expr ")"

Lexical Rules

INT := [0-9]+ IDENT := [a-zA-Z_] [a-zA-Z0-9_]* KEYWORD := "var" | "input" | "output" COMMENT := "/*" (.)* "*/" WHITESPACE:= skipped
